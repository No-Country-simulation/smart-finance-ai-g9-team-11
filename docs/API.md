# API Documentation

## Overview

The Finance AI backend exposes REST endpoints responsible for receiving financial information from the client application, processing the request, communicating with the Machine Learning service, and returning the financial analysis.

Base URL (development):

```
http://localhost:8080
```

---

# Endpoints

## Analyze Financial Profile

Performs a financial analysis based on the user's financial information.

### Request

**POST**

```
/analise-financeira
```

---

## Request Body

```json
{
  "renda_mensal": 4500,
  "nivel_endividamento": 25,
  "frequencia_poupanca": "MEDIA",
  "transacoes": [
    {
      "descricao": "Supermercado",
      "valor": 420
    },
    {
      "descricao": "Combustível",
      "valor": 300
    },
    {
      "descricao": "Streaming",
      "valor": 40
    }
  ]
}
```

---

## Request Fields

| Field               | Type                     | Required | Description                     |
|---------------------|--------------------------|----------|---------------------------------|
| renda_mensal        | BigDecimal               | Yes      | User's monthly income.          |
| nivel_endividamento | Integer                  | Yes      | User's debt level percentage.   |
| frequencia_poupanca | SavingFrequency          | Yes      | User's savings frequency.       |
| transacoes          | List<TransactionRequest> | Yes      | List of financial transactions. |

---

## Transaction Object

| Field     | Type       | Required | Description              |
|-----------|------------|----------|--------------------------|
| descricao | String     | Yes      | Transaction description. |
| valor     | BigDecimal | Yes      | Transaction amount.      |

---

## Response

**HTTP 200**

```json
{
  "perfil_financeiro": "EM_OBSERVACAO",
  "probabilidade": 0.82,
  "resumo_gastos": {
    "alimentacao": 420,
    "transporte": 300,
    "entretenimento": 40
  },
  "recomendacoes": [
    "Monitorar gastos recorrentes de entretenimento.",
    "Aumentar a reserva financeira mensal."
  ]
}
```

---

## Response Fields

| Field             | Type                    | Description                                    |
|-------------------|-------------------------|------------------------------------------------|
| perfil_financeiro | FinancialProfile        | Financial profile predicted by the AI model.   |
| probabilidade     | BigDecimal              | Prediction confidence score.                   |
| resumo_gastos     | Map<String, BigDecimal> | Total spending grouped by category.            |
| recomendacoes     | List<String>            | Financial recommendations generated by the AI. |

---

# Enumerations

## SavingFrequency

Possible values:

```
BAIXA
MEDIA
ALTA
```

---

## FinancialProfile

Possible values:

```
SAUDAVEL
EM_OBSERVACAO
ENDIVIDADO
```

---

## TransactionCategory

Categories currently supported by the Machine Learning model.

```
ALIMENTACAO
TRANSPORTE
ENTRETENIMENTO
MORADIA
SAUDE
EDUCACAO
LAZER
OUTROS
```

> The transaction category is **not** provided by the client. It is automatically inferred by the Machine Learning model based on the transaction description.

---

# Validation Rules

The backend validates all incoming requests before processing.

### Monthly Income

- Required
- Must be greater than or equal to zero

---

### Debt Level

- Required
- Must be greater than or equal to zero

---

### Saving Frequency

- Required
- Must be one of the values defined in `SavingFrequency`

---

### Transactions

- Cannot be null
- Each transaction must contain:
  - description
  - value

---

# HTTP Status Codes

| Status | Description                     |
|--------|---------------------------------|
| 200    | Analysis completed successfully |
| 400    | Invalid request                 |
| 404    | Resource not found              |
| 500    | Internal server error           |

# Processing Flow

```
Client
   │
   │ POST /analise-financeira
   ▼
Spring Boot
   │
   ▼
Request Validation
   │
   ▼
FinancialAnalysisService
   │
   ▼
FastAPI (Machine Learning)
   │
   ▼
Prediction
   │
   ▼
FinancialAnalysisResponse
```
