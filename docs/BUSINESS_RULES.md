# Business Rules

## Overview

This document describes the business rules implemented by the Finance AI platform.

The objective of the system is to analyze a user's financial behavior using Machine Learning and provide personalized financial insights based on the information provided.

---

# Business Flow

The financial analysis follows the process below.

```
User

    │

    ▼

Submit Financial Information

    │

    ▼

Backend Validation

    │

    ▼

Store Analysis Request

    │

    ▼

Send Data to Machine Learning

    │

    ▼

Machine Learning Prediction

    │

    ▼

Store Analysis Result

    │

    ▼

Return Response
```

---

# Financial Analysis

A financial analysis consists of evaluating a user's financial situation based on the following information:

- Monthly income
- Debt level
- Saving frequency
- Financial transactions

These data are processed by the Machine Learning model to generate a financial profile.

---

# Request Validation

Before processing the analysis, the backend validates the request.

Validation rules include:

- Required fields
- Numeric values greater than or equal to zero
- Valid enumeration values
- Valid transaction list

Requests that fail validation are rejected before reaching the Machine Learning service.

---

# Transaction Classification

The backend **does not classify financial transactions**.

Transaction classification is performed exclusively by the Machine Learning service.

Example:

```
Input

"Supermarket"

↓

Machine Learning

↓

ALIMENTACAO
```

This decision centralizes all classification logic in the ML model, allowing improvements without changing the backend.

---

# Spending Summary

The spending summary is generated by the Machine Learning service.

After classifying all transactions, the model groups transaction values by category.

Example:

Transactions

```
Supermarket     420

Restaurant      80

Fuel            300
```

Generated summary

```
ALIMENTACAO → 500

TRANSPORTE → 300
```

The backend only receives and returns this information.

---

# Financial Profile

The financial profile is predicted exclusively by the Machine Learning model.

Possible values:

```
SAUDAVEL

EM_OBSERVACAO

ENDIVIDADO
```

The prediction considers multiple financial variables simultaneously.

The backend never calculates the financial profile.

---

# Prediction Confidence

Along with the financial profile, the model returns a confidence score.

Example

```
Profile

EM_OBSERVACAO

Confidence

0.82
```

This value represents the confidence level of the prediction.

---

# Financial Recommendations

Recommendations are generated dynamically by the Machine Learning model.

Example

```
Increase monthly savings.

Reduce entertainment expenses.
```

Recommendations are generated according to the predicted financial profile.

The backend does not create or modify recommendations.

---

# Persistence

The backend is responsible for persisting financial analyses.

The following information is stored:

- User financial information
- Financial profile
- Prediction confidence
- Transactions
- Analysis date

Recommendations are currently **not persisted**.

---

# Saving Frequency

Saving frequency is provided by the user.

Possible values:

```
BAIXA

MEDIA

ALTA
```

The backend validates the value before sending it to the Machine Learning service.

---

# Debt Level

Currently, the debt level is provided by the client application.

The backend validates the received value and forwards it to the Machine Learning service.

Future versions may calculate this value automatically based on the user's debts.

---

# Transaction Categories

Transaction categories are inferred automatically by the Machine Learning model.

Current categories include:

- ALIMENTACAO
- TRANSPORTE
- MORADIA
- SAUDE
- EDUCACAO
- ENTRETENIMENTO
- LAZER
- OUTROS

The client application never sends transaction categories.

---

# Backend Responsibilities

The backend is responsible for:

- Validating requests
- Persisting analyses
- Calling the Machine Learning service
- Returning responses
- Managing database versioning with Flyway

The backend is **not responsible** for financial predictions.

---

# Machine Learning Responsibilities

The Machine Learning service is responsible for:

- Classifying transactions
- Predicting the financial profile
- Calculating prediction confidence
- Generating recommendations
- Building the spending summary

---

# Database Rules

Database schema changes must always be performed through Flyway migrations.

Manual changes to production databases are not recommended.

---

# Error Handling

If the Machine Learning service is unavailable:

- The backend must return an appropriate HTTP error.
- The failure should be logged.
- No incomplete analysis should be persisted.

---

# Future Business Rules

Future versions of the platform may include:

- User authentication
- Analysis history
- Financial goals
- Budget planning
- Investment suggestions
- Automatic debt level calculation
- Expense trend analysis
- Multiple Machine Learning models
